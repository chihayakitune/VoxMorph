# VoxMorph 設計書

リアルタイム波形変換型ボイスチェンジャー。Logic Pro「Vocal Transformer」の上位互換を目指す。

## 1. Vocal Transformer はなぜ音が違うのか(推定)

決定的な証拠として、Logic Pro 9 マニュアルに次の記述がある:
「The Vocal Transformer effect algorithm is based on granular synthesis」。
拡張パラメータに Grain Size が存在することもこれを裏付ける。

つまり Vocal Transformer は **時間領域のピッチ同期グラニュラー方式(PSOLA 系)** であり、これが音の性格を決めている。

- 市販ボイスチェンジャーの多くは **フェーズボコーダ(STFT)方式**。周波数領域で位相を操作するため、位相の滲みによる「水っぽさ」「金属的な残響感」、トランジェント(子音)の鈍りが避けられない。
- PSOLA 系は声門パルス1個分の波形(グレイン)を**無加工のまま**切り出し、間隔を変えて並べ直すだけ。波形そのものが保存されるので、乾いた直接的な音になり、子音も鈍らない。
- ピッチ変更=グレイン間隔の変更(スペクトル包絡=フォルマントは自動的に保存される)。フォルマント変更=グレイン内容のリサンプル。**構造上ピッチとフォルマントが最初から分離している**。後付けの包絡補正より自然。
- 破綻するときの音が「ブザー/ロボット的」で音楽的に使える(フェーズボコーダの破綻は「水没した」音)。VT の Robotize はこの性質の意図的な活用。
- モノフォニック(単声)専用に割り切っており、汎用ピッチシフタの妥協がない。
- 遅延が小さい(FFT の大窓が不要)。

本プロジェクトはこの推定に基づき、同系統の TD-PSOLA を自前実装した。

## 2. アーキテクチャ

```
入力 → リングバッファ → YINピッチ検出(f0, 有声判定)
                       → ピッチ同期グレイン切り出し(2周期・Hann窓・ピーク整列)
                       → グレインリサンプル(= フォルマント比)
                       → 新間隔で overlap-add(= ピッチ比 / Robot固定周期)
                       → 正規化バッファで振幅平坦化 → wet/dry ミックス → 出力
```

- コアは `dsp/PsolaEngine.h`(依存ゼロの単一ヘッダ)。プラグイン(JUCE)、将来のスタンドアロン、テストで共用。
- レイテンシ 2048 サンプル(48kHz で約 43ms)。ホストに報告済み。v0.2 で短縮予定。
- CPU 負荷: 実測 約2.7%(1コア, 48kHz)。

## 3. 個別フォルマント操作(F1/F2/F3…)の設計

このアイデアはリアルタイムプラグインとしてほぼ前例がなく、**本機の最大の差別化点になり得る**。実現可能。

方式(v0.3 で実装予定):

1. **分析**: 有声区間で LPC(次数14〜18、12kHzにダウンサンプルして実行)。極の周波数・帯域幅からF1/F2/F3を抽出し、時間方向に連続性トラッキング(急変を抑制)。
2. **変形**: グレイン単位でスペクトル包絡を区分ワーピング。F1/F2/F3 それぞれの近傍の包絡を目標周波数へ移動し、ゲインを個別に増減。残差(声帯音源成分)×変形後包絡で再合成。全体フォルマント(現行ノブ)への加算として動作。
3. **UI**: 各フォルマントに Shift(±st)/ Gain(dB)、トラッキング表示(F1/F2/F3 の現在値をリアルタイム描画)。

リスクと対策: フォルマント誤追跡(→帯域幅と連続性で棄却、失敗時は全体シフトへフォールバック)、F1/F2 交差時の包絡破れ(→ワーピング関数を単調に制約)、子音区間(→有声区間のみ適用しクロスフェード)。

## 5. 仮想オーディオ(OBS連携)の方針

- **Mac**: 短期は BlackHole(オープンソースの仮想デバイス)へ出力し OBS で受ける。長期は BlackHole と同じ AudioServerPlugIn 方式の自前ドライバを同梱(参考実装が公開されており移植性が高い)。
- **Windows**: 短期は VB-Cable 連携。長期は自前 WDM 仮想ドライバ(署名コストが高いため、まず既存デバイス連携で出荷)。
- アプリ側は「出力デバイス選択」を実装すれば両OSとも即対応できる(JUCEの標準機能)。
